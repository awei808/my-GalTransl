## 一、核心问题：配置文件名 `config.yaml` 写死

共发现 **11 处** 写死 `config.yaml`（含 API 默认参数与请求体）。

### 后端对该字段的处理分两类（这是判定的关键）

| 类别 | 接口 | 行为 |
|---|---|---|
| **有自动回退（安全）** | `submitJob`（server.py:68-70）、`check-model`（上次修复已加回退） | 传入 `config.yaml` 时若文件不存在，后端按 `config.inc.yaml` → `config.yaml` 回退解析 |
| **无回退（bug）** | `GET /config`（server.py:1190）、`PUT /config`（server.py:2808）、`GET /dictionary`（server.py:1835）、`GET /dictionary/project`（server.py:1881） | 直接用传入名拼路径，文件不存在即 404，无回退 |

> 另外 `POST /cache/save` 上次已修复：配置名不存在时回退探测 `config.inc.yaml` → `config.yaml`（server.py:1347 附近）。

### 逐处判定

| 位置 | 用途 | 后端是否回退 | 结论 |
|---|---|---|---|
| `lib/api/project.ts:37` `fetchProjectConfig` 默认参数 | 读项目配置 | 否（GET） | ❌ bug：config.inc.yaml 项目 404 |
| `lib/api/project.ts:101` `saveCacheFile` 兜底 | 保存缓存 | 是（已修） | ✅ 安全 |
| `lib/api/project.ts:179/185` 字典读取默认 | 读字典 | 否（GET） | ❌ bug |

| `pages/wizard/NewProjectWizard.tsx:287/323` 保存 | 新建项目保存 | 新建本就是 config.yaml | ✅ 正确默认 |

### 根因

前端 `appStore` **没有** “当前项目真实配置文件名” 字段；`openProject(projectId)` 只设 `activeProjectId`，不查历史、不调后端探测。因此所有页面只能假设 `config.yaml`。

## 三、是否需要改代码？结论

- **必须改（仅 config.inc.yaml 项目受影响，而你的真实翻译项目正是 config.inc.yaml）**：
  - `ProjectConfigPage`（项目设置页读取/保存）
  - `DictionaryPage`（字典读取/保存）
  - `fetchProjectConfig` / `fetchProjectDictionary` / `fetchProjectDictionaryManager` 等 GET 调用
  - 这些在 config.inc.yaml 项目下会 **404 或写错文件**。
- **不用改**：`submitJob`、`checkModelAvailability`、新建项目向导（后端有回退或本就正确）、后端地址端口。

---

## 四、修复建议（二选一）

### 方案 A（推荐，最小改动、低风险）：后端统一回退
新增 `_resolve_config_file(project_dir, requested)` 助手，在 `GET /config`、`PUT /config`、`GET /dictionary`、`GET /dictionary/project` 中复用 submit 已有的回退逻辑。此后端一旦修复，前端 `config.yaml` 默认即“全局生效”，**前端无需改动**即可消除 config.inc.yaml 隐患。

### 方案 B（更干净但侵入大）：前端贯通真实配置名
1. 后端新增/复用接口暴露自动探测到的 `config_file_name`（如 `GET /files` 或 `/meta` 返回）。
2. `appStore` 增加 `activeConfigFileName` 字段，`openProject` 时写入（数据源：历史记录或后端 meta）。
3. 所有 API 调用用该字段替换字面量 `config.yaml`。
- 需新增数据源，改动面大。

> 建议先走 **方案 A** 彻底消除隐患；方案 B 可后续作为“界面显示真实配置名”的增强项。

## 五、附：当前已知 config.inc.yaml 项目受影响的功能清单
- 项目设置页：打不开配置 / 保存写错文件 → 直接影响
