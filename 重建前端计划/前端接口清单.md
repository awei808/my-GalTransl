# 前端接口清单（Tauri IPC + HTTP API）

> 基于保留接口层 `lib/api/` 实测导出，含 2026-07-17 补完的 3 个新接口。
> 重构后新前端（SolidJS）**可原样引用**此清单中的所有接口，无需修改后端。

---

## 一、Tauri Rust 命令（IPC `invoke`，非 HTTP）

| 命令名 | 参数 | 说明 | 前端调用场景 |
|---|---|---|---|
| `ensure_backend_ready` | `{hideConsole, timeoutMs}` | 检测 12333 端口，未监听则拉起 Python 后端（源码或 exe） | 首次连接 / 断线重连 |
| `open_folder` | `{path}` | 用系统资源管理器打开目录 | "打开项目文件夹"菜单 |
| `reveal_file` | `{path}` | 在资源管理器定位并高亮文件 | 问题列表点击"定位文件" |
| `create_dir` | `{path}` | 创建目录 | 新建字典文件等 |
| `write_text_file` | `{path, content}` | 写文本文件 | 保存本地配置 |
| `copy_files` | `{sources[], dest}` | 批量复制文件 | 文件导出 |

全部在 `desktop/src-tauri/src/main.rs` 注册，capabilities `default.json` 授权。新前端直接 `invoke('命令名', {...})` 调用，框架无关。

## 二、HTTP API — 按新前端页面分类

### 2.2 翻译控制台（需项目已打开）

| 函数 | HTTP | 说明 | 轮询？ |
|---|---|---|---|
| `fetchProjectProgress(projectId)` | `GET /api/projects/:id/progress` | 翻译进度（total/translated/problems/failed，按文件拆分） | ⬜ |
| `fetchProjectRuntime(projectId)` | `GET /api/projects/:id/runtime` | 运行时快照：stage、stage_index/total、翻译速度、ETA、最近错误/成功、**实时提示词预览**、**实时译文拼接预览** | ✅ 建议 2~3s 间隔 |
| `submitJob(payload)` | `POST /api/jobs` | 提交翻译任务（选择后端 profile + 项目目录 + 引擎） | — |
| `stopProjectTranslation(projectId)` | `POST /api/projects/:id/stop` | 停止当前翻译任务 | — |

> **注意**：`fetchProjectRuntime` 在运行时建议以 2-3 秒间隔轮询。
> `stage_index`/`stage_total` 为 2026-07-17 新增字段，驱动"流程 7 阶段完成情况"显示。
> `latest_prompt_preview`/`latest_assembled_preview` 为同日新增，驱动"当前提示词"/"译文拼接"两栏。

### 2.3 校对审核（需项目已打开 + 文件已选中）

#### 文件浏览

| 函数 | HTTP | 说明 |
|---|---|---|
| `fetchProjectFiles(projectId)` | `GET /api/projects/:id/files` | 项目文件树：`input_files[]` / `output_files[]` / `cache_files[]`，各含 name/size/modified/entry_count |

#### 翻译缓存读写（CacheEntry JSON 数组）

| 函数 | HTTP | 说明 |
|---|---|---|
| `fetchProjectCache(projectId)` | `GET /api/projects/:id/cache` | 所有缓存文件列表 |
| `fetchCacheFile(projectId, filename)` | `GET /api/projects/:id/cache/:filename` | 读取单个缓存文件，返回 `entries: CacheEntry[]`（18 字段） |
| `saveCacheFile(projectId, filename, entries)` | `POST /api/projects/:id/cache/save` | 保存整个缓存文件（全量覆盖）。校对审核增删改条目后调用 |
| `deleteCacheEntry(projectId, filename, index)` | `POST /api/projects/:id/cache/delete-entry` | 按 index 删除单条缓存 |
| `deleteCacheFiles(projectId, filenames[])` | `POST /api/projects/:id/cache/delete-file` | 批量删除缓存文件 |

#### 校对审核属性面板字段<br>（基于 `CacheEntry` 的 18 个字段）
`index` `name` `pre_src` `post_src` `pre_dst` `proofread_dst` `trans_by` `proofread_by` `problem` `trans_conf` `doub_content` `unknown_proper_noun` `pre_jp` `post_jp` `pre_zh` `proofread_zh` `post_zh_preview` `post_dst_preview`

## 四、协议细节

- **HTTP base URL**：`http://127.0.0.1:12333`（由 `getBackendBaseUrl()` 管理，桌面托管态由 `ensureDesktopBackendReady` 的动态端口覆盖）
- **项目 ID 编码**：`encodeProjectDir(absolutePath)` → Base64（用在 URL 路径中），`decodeProjectDir(token)` 反向
- **请求体格式**：JSON（`Content-Type: application/json`），`saveCacheFile` / `updateProjectConfig` 等写操作用 `orjson`
- **CORS**：已设 `Access-Control-Allow-Origin: *`（`server.py` 892 行）
- **404/错误**：返回 `{"error": "message"}`，HTTP 状态码 4xx/5xx
- **流式接口**：仅 `getAiTranslateUrl` 为流式 text/plain，需用 `fetch().body.getReader()` 读

## 五、新前端（SolidJS）集成方式

1. 将 `重建前端计划/保留文件/src/lib/api/` 整目录拷贝到新 Solid 工程的 `src/lib/api/`
2. 在新 `package.json` 加入依赖：`@tauri-apps/api`（用于 `invoke`、`listen`）——这是 `client.ts` 的唯一外部依赖
3. `lib/api` 层为零框架绑定（已验证 0 处 react/zustand 引用），Solid 中直接 `import { fetchProjectRuntime } from '@/lib/api'` 使用
4. 14 个偏好函数（localStorage 系）完全无依赖，跨框架零成本搬运
