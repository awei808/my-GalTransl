# 吐司弹窗 & 居中确认弹窗 — 重构规格

> 重构栈：SolidJS
> 配色来自「更新计划」第 4 条：浅色主色 #0066CC / 深色主色 #0099FF / 浅色填充 #F0F5FF / 深色填充 #12161C
> 圆角遵循 8px 网格系统（弹窗 12px = 1.5×8px，按钮 6px = 0.75×8px）

---

## 一、右下角吐司弹窗（Toast / Popup）

### 1.1 参考来源要点（`popup.js` 中的 `PopupManager`）

| 特性 | 参考值 | 重构后 |
|---|---|---|
| 定位容器 | `position:fixed; bottom:20px; right:20px; z-index:1000` | **同右**，safe-area 兜底 |
| 容器布局 | `flex-direction:column-reverse; gap:10px`（新 toast 在最上方） | **同右** |
| 最大数量 | 3 个，超出挤掉最旧 | **同右**，Map 插入序 = 最旧 |
| 自动消失 | 2.5s | **6s**（用户要求），可通过 `duration` 参数覆盖 |
| 类型 | `error`(红) / `warning`(橙) / `success`(绿) | **同右**，配色改写为 #0066CC 体系 |
| 淡出动画 | `slideOutDown 0.3s ease-in` | **同右**（方向改 bottom→bottom） |
| 入场动画 | `slideInUp 0.3s ease-out` | **同右** |
| 关闭按钮 | `×` 字符按钮，`pointerup` 事件 | **改为 SVG 图标**（`×` 圆形关闭），`pointerup` |
| HTML 转义 | `escapeHtml()` via DOM `textContent` | **同右**；若显式允许 HTML 则走 sanitize |
| 容器创建 | 惰性 `createPopupBox()` 附加到 `document.body` | **Solid `<Portal>` 挂 `document.body`** |

### 1.2 SolidJS 实现方案

**核心：模块级 store + `<ToastHost>` 单例**

```ts
// stores/toastStore.ts
import { createStore } from 'solid-js/store';

export type ToastTone = 'error' | 'warning' | 'success' | 'info';

export interface ToastEntry {
  id: string;
  message: string;
  tone: ToastTone;
  duration: number;          // ms, default 6000
  allowHtml?: boolean;       // 是否允许 HTML（须 DOMPurify）
  createdAt: number;
}

const [toasts, setToasts] = createStore<{ items: ToastEntry[] }>({ items: [] });
const MAX_TOASTS = 3;

let _counter = 0;
function uid() { return `t${Date.now()}-${++_counter}`; }

// 公开 API
export const toast = {
  show(message: string, opts?: { tone?: ToastTone; duration?: number; allowHtml?: boolean }) {
    const id = uid();
    const entry: ToastEntry = {
      id,
      message,
      tone: opts?.tone ?? 'info',
      duration: opts?.duration ?? 6000,
      allowHtml: opts?.allowHtml ?? false,
      createdAt: Date.now(),
    };
    setToasts('items', items => {
      const next = [...items, entry];
      if (next.length > MAX_TOASTS) next.shift(); // 挤掉最旧
      return next;
    });
    return id;
  },

  dismiss(id: string) {
    setToasts('items', items => items.filter(t => t.id !== id));
  },

  success(msg: string, duration?: number) { return toast.show(msg, { tone: 'success', duration }); },
  error(msg: string, duration?: number)   { return toast.show(msg, { tone: 'error', duration }); },
  warning(msg: string, duration?: number) { return toast.show(msg, { tone: 'warning', duration }); },
  info(msg: string, duration?: number)    { return toast.show(msg, { tone: 'info', duration }); },
};
```

**`<ToastHost>` 组件**（挂于 App 根节点一次，Solid `<Portal>` 到 body）

```tsx
<Portal>
  <Show when={toasts.items.length > 0}>
    <div class="toast-host">
      <For each={toasts.items}>
        {(t) => <ToastItem entry={t} />}
      </For>
    </div>
  </Show>
</Portal>
```

**每个 `<ToastItem>`**：
- 渲染时启动 `setTimeout` → `toast.dismiss(id)`
- 手动点 × → `toast.dismiss(id)`
- 应用 `fade-out` class 提前 300ms（留给动画）
- `pointerup`（非 `click`，避免移动端 300ms 延迟）

### 1.3 配色（对齐 #0066CC 双主题体系）

### 1.4 与参考实现的关键差异

| 项目 | 参考 | 重构 |
|---|---|---|
| 默认超时 | 2.5s | **6s**（用户要求），可参数覆盖 |
| 状态管理 | `Map<string, HTMLElement>` + 手动 DOM | Solid `createStore` 响应式，声明式渲染 |
| 关闭图标 | `×` 字符 | **SVG 圆角关闭图标**（遵守禁 emoji 规则） |
| 容器挂载 | 惰性 `createPopupBox()` | **Solid `<Portal>`** 在 App 初始化时挂载 |
| HTML 支持 | `escapeHtml` 全部转义 | 默认转义，设置 `allowHtml:true` + **DOMPurify** |
| DOM 复用 | 每次创建新 div | Solid `<For>` 自动 diff + 清理 |

## 二、居中确认弹窗（Confirm Dialog）

### 2.1 参考来源要点（`popup.js` 中的 `PopupCentreManager`）

| 特性 | 参考值 | 重构后 |
|---|---|---|
| 遮罩层 | `position:fixed` 全屏 `rgba(0,0,0,0.5)`, z-index:2000 | **同右** |
| 弹窗定位 | `top:50%; left:50%; transform:translate(-50%,-50%)`, 白底圆角 12px | **同右**，底色改为 `#F0F5FF`（浅色）/ `#12161C`（深色） |
| 入场动画 | `scale(0.9)→scale(1)` + `opacity:0→1`, 0.3s ease | **同右** |
| 结构 | 遮罩 `.show` 先行 + 10ms 后弹窗 `.show`（分层动画） | **同右** |
| 按钮 | 取消(灰)+确认(绿) | **确认主色 #0066CC, 取消次要灰** |
| 键盘 | Enter=确认, Escape=取消, once 监听+preventDefault | **同右** |
| 自动聚焦 | 有输入框→聚焦输入框；无→聚焦确认按钮 | **同右** |
| 嵌入 HTML | `attachment:'input'` 加 `<input>` | 扩展为 `html: string` 参数，用 `innerHTML` + DOMPurify |
| 单例 | 新弹窗打开时关闭旧弹窗 | **同右** |

### 2.2 SolidJS 实现方案

**Promise-based API**（调用方等结果，不传 callback）

```ts
// stores/confirmStore.ts
interface ConfirmOptions {
  title: string;                       // 标题
  message: string;                     // 正文（纯文本）
  html?: string;                       // 可选 HTML 内容（取代 message，须 DOMPurify）
  inputLabel?: string;                 // 可选输入框标签
  inputPlaceholder?: string;           // 可选输入框占位符
  inputDefault?: string;               // 可选输入框默认值
  confirmText?: string;                // 确认按钮文案，默认"确认"
  cancelText?: string;                 // 取消按钮文案，默认"取消"
  tone?: 'danger' | 'warning' | 'info' | 'default'; // 确认按钮风格
}

interface ConfirmResult {
  confirmed: boolean;
  inputValue?: string;                 // inputLabel 不为空时返回输入值
}

export const confirm = {
  show(options: ConfirmOptions): Promise<ConfirmResult> { /* ... */ },
};
```

**调用示例**

```ts
// 简单确认
const ok = await confirm.show({
  title: '删除确认',
  message: '此操作不可恢复，确定删除吗？',
  tone: 'danger',
});

// 带输入框
const result = await confirm.show({
  title: '修改字段名',
  message: '请输入新的字段名：',
  inputLabel: '字段名',
  inputPlaceholder: '如 proofread_dst',
  inputDefault: currentField,
});

// 嵌入 HTML（from 用户需求："弹窗内部可嵌入html"）
const ok = await confirm.show({
  title: '翻译不一致',
  html: '<p>原文：<code>こんにちは</code></p><p>已有译文：<em>你好</em></p><p>本次译文：<em>您好</em></p>',
  tone: 'warning',
});
```

**`<ConfirmHost>` 组件**（单例，一次只显示一个）

- 用 Solid `<Portal>` 渲染到 `document.body`
- 管理内部状态：`visible`, `options`, `resolve`
- 当 `confirm.show()` 被调用时：
  1. 如果已有活动弹窗，先 `resolve({confirmed:false})` 关闭旧弹窗
  2. 设置新 options + visible=true
  3. 打开遮罩 → 10ms 后打开弹窗（分层动画）
  4. 用户按确认/取消/Escape → `resolve(...)` + 关闭动画 → 销毁
- 若 `tone='danger'`，确认按钮用红色系
- 若 `tone='warning'`，确认按钮用橙色系  
- 若 `tone='info'` 或 `'default'`，确认按钮用 #0066CC

### 2.3 安全：嵌入 HTML 的 XSS 防护

用户要求"弹窗内部可嵌入 html"。必须做：

1. **前端 clean**：所有 HTML 内容经过 **DOMPurify**（`npm i dompurify && npm i -D @types/dompurify`）
2. **Solid 渲染**：`<div innerHTML={DOMPurify.sanitize(html)} />` 注意：Solid 的 `innerHTML` 属性本身不做转义，必须手动 sanitize
3. **后端信任边界**：即使后端返回的 HTML 也要 sanitize（防御原则）

### 2.4 确认弹窗的底色与按钮色调

**与更新计划设计体系对齐**：弹窗背景使用填充色，按钮主色匹配品牌色。

**弹窗容器**：
```css
:root {
  --confirm-bg:       #F0F5FF;   /* 浅色填充 */
  --confirm-text:     #333;
  --confirm-cancel-bg: #e5e7eb;
  --confirm-cancel-text: #374151;
  --confirm-danger:   #dc2626;
  --confirm-warning:  #d97706;
  --confirm-brand:    #0066CC;
}
:root[data-theme='dark'] {
  --confirm-bg:       #12161C;   /* 深色填充 */
  --confirm-text:     #e5e7eb;
  --confirm-cancel-bg: #374151;
  --confirm-cancel-text: #d1d5db;
  --confirm-danger:   #ef4444;
  --confirm-warning:  #f59e0b;
  --confirm-brand:    #0099FF;
}
```

**确认按钮色调映射**（浅色 / 深色相同语义）：

```
tone='danger'  → 确认按钮 bg: var(--confirm-danger), color: #fff
tone='warning' → 确认按钮 bg: var(--confirm-warning), color: #fff
tone='info'    → 确认按钮 bg: var(--confirm-brand), color: #fff
tone='default' → 确认按钮 bg: var(--confirm-brand), color: #fff
取消按钮始终: bg: var(--confirm-cancel-bg), color: var(--confirm-cancel-text)
```

**遮罩层**：`rgba(0,0,0,0.5)` 统一不变（浅/深色均适用）。

### 2.5 与更新计划的对齐点

- **dismissible 行为**：默认点遮罩 = 取消（类 VSCode 对话框行为），与 `popup.js` 参考一致；`tone='danger'` 时 `dismissible: false`（阻止误触）
- **键盘不冲突**：Enter/Escape 在弹窗激活时 `preventDefault` + `stopPropagation`，不影响全局 ctrl+z/x/c/v/f/h/a/s 的语义
- **"弹窗内部可嵌入 html"**（更新计划第 18 条）：已实现为 `html` 参数 + DOMPurify；确认弹窗的位置是"在缺少非关键数据时向用户确认是否执行操作"

### 2.6 与原 React 版 `ConfirmDialog` 的迁移对照

| 项目 | 原 React 版 | 新 Solid 版 |
|---|---|---|
| 驱动方式 | `useUIStore.confirm()` → Promise | **同**（API 兼容，迁移量最小） |
| 选项参数 | `{title, message, confirmText, cancelText, dismissible, tone}` | **扩展**：新增 `html`, `inputLabel`, `inputPlaceholder`, `inputDefault` |
| dismissible | 点遮罩可关 | **保留**（点遮罩=取消）；danger 时 `dismissible:false` |
| 挂载方式 | `createPortal` in App.tsx | **Solid `<Portal>`** |
| HTML 渲染 | 不支持 | **`innerHTML` + DOMPurify** |
| 输入框 | 不支持 | **`inputLabel` 参数**（`PopoupCentreManager.attachment='input'` 的移植） |

## 三、两个组件在 App 中的挂载

Solid `<Portal>` 确保始终渲染在 `<body>` 末端、不受主界面 z-index 影响。

```tsx
// App.tsx (Solid)
import { ToastHost } from './components/toast/ToastHost';
import { ConfirmHost } from './components/confirm/ConfirmHost';

export const App = () => (
  <>
    <Router>...</Router>
    <ToastHost />        {/* 右下角吐司容器，z-index: 1000 */}
    <ConfirmHost />      {/* 居中确认弹窗容器，z-index: 2000 */}
  </>
);
```

两者均为零 props 的独立组件，内部完全依赖 module-level store。
任何页面/组件直接 `import { toast }` 或 `import { confirm }` 使用，无需穿透 props。

> **依赖**：吐司和确认弹窗的颜色从 `tokens.css` 的 CSS 变量取值（`--toast-*` / `--confirm-*`）。
> `tokens.css` 在 `<App>` 初始化时引入，包含 `:root` 和 `:root[data-theme='dark']` 两套变量。
> 主题切换通过 `<html data-theme>` 属性驱动（由用户"设置"界面控制，`setThemeModePreference()` 调用 localStorage）。
