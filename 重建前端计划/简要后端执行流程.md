> ⚠️ **文档状态（2026-07-23 修订）**：本文件为重建前的后端流水线概览，与当前 `GalTransl/` 实现基本一致（阶段 0–3 对应 `ForGlobalPrompt`/`GenDic`/`ForFileMetaData`/`ForBatchMetaData`，阶段 6 对应 `ForGalJsonMulitChat` + `BaseTranslate`）。`CacheEntry` 字段改用规范命名 `pre_dst`/`proofread_dst`。**不**补充"BatchMetadata 注入局限/稀释"说明——该机制为设计内的文件/块级风格引导，非缺陷。

阶段 0: 新建翻译项目，录入数据文件夹，输入数据校验
  读取 input/ 目录下所有 .json 文件（原文文件）
  逐条校验结构完整性

阶段 1: 文本无损压缩
  TextCompressor 压缩json格式为普通对话格式，减少 token 消耗

阶段 2: 生成全局游戏分析
  ForGlobalPrompt.batch_translate() → 调用 LLM
  生成游戏世界观、角色关系、特殊用语等全局分析，

阶段 3: 构建术语表
  GenDic.batch_translate() → 调用 LLM
  生成译前/GPT/译后/人名替换，四套词典文件（可跳过如果使用人工编辑）

阶段 4: 生成文件级元数据
  ForFileMetaData.batch_translate() → 逐文件调用 LLM
  分析每文件的剧情上下文、角色身份

阶段 5: 划分翻译区间
  ForBatchMetaData.batch_translate() → 逐文件调用 LLM
  将每文件原文按语义/剧情拆分为翻译批次（batch）

阶段 6: 翻译执行（核心耗时阶段）
  _run_translation_phase() → 对每个文件：
    1. 按批次切分原文 chunks
    2. worker 协程池并发翻译
    3. 每个 batch：
       a. ForGalJsonMulitChat.translate()
            → _build_round_user_content()  ← 组装 LLM 提示词
            → _call_llm()                   ← 调用 LLM API（流式/非流式）
            → _parse_stream_lines()         ← 解析流式响应
            → _handle_parse_result()        ← 提取译文写入 CSentense
       b. save_transCache_to_json()         ← 每 save_steps 批次写缓存文件
    4. 全部文件完成后 → 输出到缓存目录并进行译文问题检测

阶段 7: 校对审核阶段（人工 + 后端协作）

  读取 cache/ 目录下所有 .json 缓存文件
  显示每条 CacheEntry 的 18 个字段

  人工校对流程（UI 驱动）：
  1. 用户从左侧文件树选择缓存文件
  2. 加载该文件 → 以卡片列表展示每条条目
  3. 对每条条目可执行以下操作：
     a. 修改 pre_dst（机器翻译结果）
     b. 填写 proofread_dst（人工校对译文）
     c. 标记/清除 problem 问题类型
     d. 调整 trans_conf 置信度
     e. 删除整条条目（触发局部重翻）
  4. 修改标记 dirty → 用户点"保存"触发全量覆写
     saveCacheFile(projectId, filename, entries)
       → 后端全量覆写 cache/filename.json
       → 前端标记 clean
  5. 可选：跨文件全文搜索/替换
     searchCache(projectId, query, field)
       → 跨所有缓存文件搜索原文/译文/问题
     replaceCache(projectId, query, replacement, field, dryRun)
       → 批量替换译文（支持 dry-run 预览）

阶段 8: 构建输出阶段
  读取 cache/ 目录下已修改的 .json 缓存文件
  读取 input/ 目录下对应的原始 .json 原文文件

输出构建流程（自动 + UI 触发）：
  1. 逐文件处理：
     a. 加载 cache/<filename>.json ← 包含人工校对后的 CacheEntry[]
     b. 加载 input/<filename>.json ← 原始输入 JSON
     c. update_json_with_transList(all_trans_list, all_json_list, name_replaceDict)
          → 将 CacheEntry 中的 pre_dst / proofread_dst 等翻译结果
            合并回原始输入 JSON 的对应字段
          → 应用人名替换字典（name_replaceDict）
     d. save_func(output_file_path, final_result)
          → 合并后的完整 JSON 写入 output/gt_output/<filename>.json
          → 输出格式由文件插件决定（file_galtransl_json、file_subtitle_srt_lrc_vtt 等）
  2. 全部文件完成后 → 输出目录结构：
     output/
     └── gt_output/
         ├── 01_intro.json          ← 含翻译结果的完整 JSON
         ├── 02_chapter1.json
         └── ...

触发方式：
 手动触发：校对审核完成后，用户通过 UI 点击"构建输出"
