## 一、路由表

### 1.1 路径 → 组件映射

```
/                               → <HomePage>              无项目依赖，始终可达
/settings                       → <SettingsPage>          无项目依赖，始终可达

/project/:id/translate          → <TranslateConsole>     需已打开项目（路由守卫）
/project/:id/review             → <ReviewPage>            需已打开项目 + 已选中文件
```

左侧按钮栏（Activity Bar）激活的是**侧栏内容**，不是页面路由：

| 按钮 | 行为 | 路由影响 | 侧栏内容 |
|---|---|---|---|
| 翻译控制台 | 切换 main view | 路由变 | — |
| 校对审核 | 先切 view，再激活侧栏"文件树" | 路由变 | 文件树 |
| 查找替换 | 切换侧栏内容（主界面不变） | 路由不变 | 搜索框 |
| 问题检测 | 切换侧栏内容（主界面不变） | 路由不变 | 问题列表 |
| 设置 | 切换 main view | 路由变 | — |

### 1.2 路由守卫

```
/project/:id/* 路径需要以下条件：
  1. activeProjectId !== null
  2. fetchProjectFiles 至少成功过一次
  不满足 → redirect to /
```

### 1.3 组件结构

```
src/
  pages/
    HomePage.tsx
    SettingsPage.tsx
    translate/
      TranslateConsole.tsx       ← 主界面渲染
    review/
      ReviewPage.tsx             ← 主界面渲染
  features/
    sidebar/
      SidebarExplorer.tsx        ← 文件树（校对审核用）
      SidebarSearch.tsx          ← 查找替换
      SidebarProblems.tsx        ← 问题检测
```

## 二、中央状态 `createStore` 模型

```ts
// stores/appStore.ts
import { createStore } from 'solid-js/store';

export type ActiveView = 'home' | 'translate' | 'review' | 'settings';
export type ConnectionPhase = 'offline' | 'connecting' | 'online' | 'reconnecting';
export type SidebarTab = 'explorer' | 'find' | 'problems' | null;

export interface AppState {
  // ── 导航 ──
  activeView: ActiveView;
  sidebarOpen: boolean;
  sidebarTab: SidebarTab;        // 当前侧栏显示什么内容

  // ── 项目 ──
  activeProjectId: string | null;
  activeFilePath: string | null; // 校对审核选中的文件
  dirtyFiles: string[];          // 有未保存修改的文件路径列表

  // ── 连接 ──
  connectionPhase: ConnectionPhase;
  connectionTimeoutMs: number;   // 当前等待超时

  // ── 后端配置 ──
  backendOnline: boolean;
}

export const defaultState: AppState = {
  activeView: 'home',
  sidebarOpen: true,
  sidebarTab: null,
  activeProjectId: null,
  activeFilePath: null,
  dirtyFiles: [],
  connectionPhase: 'offline',
  connectionTimeoutMs: 20000,
  backendOnline: false,
};

export const [appState, setAppState] = createStore<AppState>(defaultState);

// ── 便捷 action（Solid createStore 不用 reducer，直接写函数）──

export function navigateTo(view: ActiveView) {
  setAppState('activeView', view);
  // view 切换时自动关闭与项目无关的侧栏
  if (view === 'settings') setAppState({ sidebarOpen: false, sidebarTab: null });
}

export function openProject(projectId: string) {
  setAppState({
    activeProjectId: projectId,
    activeView: 'translate',
    sidebarOpen: true,
    sidebarTab: 'explorer' as SidebarTab,
  });
}

export function closeProject() {
  // 复位所有项目相关状态
  setAppState({
    activeProjectId: null,
    activeView: 'home',
    sidebarOpen: false,
    sidebarTab: null,
    activeFilePath: null,
    dirtyFiles: [],
  });
}
```

### 2.1 状态流转

```
openProject(id)     → activeProjectId, activeView='translate', sidebarOpen=true
navigateTo('review')→ activeView='review', sidebarTab='explorer'（自动打开文件树）
closeProject()      → 全部复位到 defaultState（activeView='home'）
```

```

        <MainArea />                 ← 主界面（由 activeView 驱动）
      </div>

      <ConfirmHost />                ← 居中确认弹窗
    </ConnectionManager>
```

### 3.2 布局的 CSS 方案（建议）

```css
.app-body {
  display: grid;
  grid-template-columns: 48px var(--sidebar-width, 260px) 1fr;
  height: calc(100vh - var(--titlebar-height, 42px) - var(--statusbar-height, 28px));
}
```

`--sidebar-width` 支持拖拽：`<SidebarPanel>` 根 div 监听 `pointerdown` 在右侧边缘 → `pointermove` 修改 CSS 变量 → `pointerup` 释放。（**不引入外部库**，纯 pointer events + CSS 变量 ~ 80 行）

翻译控制台的两栏也同理：`grid-template-columns: 1fr 1fr` + 可拖拽分隔线。

### 3.3 ActivityBar → Sidebar 驱动逻辑

```tsx
// ActivityBar.tsx
const tabs = [
  { icon: 'translate', view: 'translate' as const, label: '翻译控制台' },
  { icon: 'review',    view: 'review' as const,    label: '校对审核' },
  { icon: 'search',    view: 'search',              label: '查找替换' },
  { icon: 'problems',  view: 'problems',            label: '问题检测' },
  { icon: 'settings',  view: 'settings' as const,   label: '设置' },
];

function handleTabClick(tab: Tab) {
  if (tab.view === 'settings') {
    navigateTo('settings');
    setAppState({ sidebarOpen: false });
  } else if (['search', 'problems'].includes(tab.view)) {
    // 切换侧栏内容，主界面不变
    setAppState({ sidebarOpen: true, sidebarTab: tab.view as SidebarTab });
  } else {
    navigateTo(tab.view as 'translate' | 'review');
    setAppState({ sidebarOpen: true, sidebarTab: tab.view === 'review' ? 'explorer' : null });
  }
}
```

### 3.4 SidebarPanel 内容随 `sidebarTab` 切换

```tsx
// SidebarPanel.tsx
<Switch>
  <Match when={sidebarTab() === 'explorer'}>  <FileExplorer />          </Match>

  <Match when={sidebarTab() === 'problems'}>  <ProblemList />           </Match>
  <Match when={!sidebarTab()}>                <EmptySidebar />          </Match>
</Switch>
```

## 四、连接状态机（解决"后端掉线怎么办"）

### 4.1 状态定义（与 appState 中 `connectionPhase` 对应）

```
            ┌──────────┐
    start ──→  offline  ─────→ connecting
            └──────────┘          │
                                  │ (timeout)
                                  ↓
                          ┌──────────────┐
                          │  connecting   │──→ online
                          └──────────────┘      │
                              ↑                 │ (轮询失败)
                              │                 ↓
                              │          ┌─────────────┐
                              └──────────│ reconnecting │
                                         └─────────────┘
                                              │     │
                                              │ (3s)│ (放弃)
                                              ↓     ↓
                                           offline
```

### 4.2 触发时机

| 事件 | 转换 |
|---|---|
| `App` 挂载 | `offline → connecting` |
| `ensureDesktopBackendReady` 成功 | `connecting → online` |
| `ensureDesktopBackendReady` 超时 | `connecting → online`（降级：允许用户手动操作） |
| 任意 HTTP 请求 4xx/5xx | 如 502/503 → `online → reconnecting` |
| 连接恢复（轮询成功） | `reconnecting → online` |
| 用户主动断开 | `→ offline` |
| 连续重试 5 次失败 | `reconnecting → offline` |

### 4.3 ConnectionManager 组件

```tsx
// ConnectionManager.tsx — 包在 <App> 内部，负责后台轮询
function ConnectionManager(props: { children: JSX.Element }) {
  const [state, setState] = useAppStore();

  onMount(async () => {
    setState('connectionPhase', 'connecting');
    try {
      await ensureDesktopBackendReady({ timeoutMs: state.connectionTimeoutMs });
      setState({ connectionPhase: 'online', backendOnline: true });
      startPolling();
    } catch {
      setState('connectionPhase', 'offline');
      // 可选：弹 toast 提示用户手动操作
    }
  });

  function startPolling() {
    setInterval(async () => {
      try {
        await fetchVersion(); // 轻量保活探测
      } catch {
        setState({ connectionPhase: 'reconnecting', backendOnline: false });
        retryConnection();
      }
    }, 10000);
  }
}
```

### 4.4 对 UI 的影响

- `connectionPhase === 'offline'` → 首页显示"未连接到翻译引擎"提示，项目相关按钮灰化/隐藏
- `connectionPhase === 'connecting'` → 按钮显示"连接中…"（按钮可用但不可点击项目相关）
- `connectionPhase === 'online'` → 正常可用
- 所有 `fetchProject*` 类调用失败时，自动触发 `reconnecting`

## 五、项目打开/关闭全链路（解决"项目怎么进来、怎么出去"）

### 5.1 打开项目

```
用户点击 文件 → 打开项目
  → Tauri `open_folder()` 对话框 → 用户选一个目录
  → 用户确认
  → setAppState('connectionPhase', 'connecting')
  → ensureDesktopBackendReady({ timeoutMs: 30000 })
       ├─ 成功 → setAppState('connectionPhase', 'online')
       └─ 失败 → toast.error('无法启动后端服务，请手动运行 run_backend.py')
                 setAppState('connectionPhase', 'offline')
                 流程继续（用户可能已手动起后端）
  → const projectId = encodeProjectDir(selectedPath)
  → try:
      fetchProjectFiles(projectId)     ← 验证项目合法性
    catch:
      toast.error('所选目录不是有效的 GalTransl 项目')
      return
  → openProject(projectId)            ← 设置状态 + 导航
  → fetchProjectRuntime(projectId)    ← 首次轮询（异步，不阻塞导航）
  → toast.success('项目已打开')
```

**细节**：
- `open_folder` 是 Tauri Rust 命令（已经在 main.rs 注册），直接用 `invoke('open_folder', { path: '' })` 调系统文件夹选择对话框
- `ensureDesktopBackendReady` 的超时建议 **30s**（而不是默认的 20s），因为 Python 冷导入可能需要 15s+
- 项目合法性检查：`fetchProjectFiles` 返回 `input_dir` / `output_dir` / `cache_dir` 齐全即认为合法

### 5.2 关闭项目

```
用户操作：
  A. 点击 文件 → 关闭项目
  B. 点击 侧栏关闭按钮（handleCloseProject 调用 closeProject）
  C. 打开新项目（隐式关闭旧项目）

流程：
  → 检查 dirtyFiles.length > 0
       ├─ 有未保存 → confirm.show({ title: '未保存的更改', message: '...', tone: 'warning' })
       │               ├─ 确认 → 丢弃更改，继续
       │               └─ 取消 → 中断关闭
       └─ 无未保存 → 继续
  → closeProject()                         ← 复位所有项目状态到 defaultState
  → navigate('/')                          ← 回首页
  → toast.info('项目已关闭')
```

### 5.3 关联的状态变化

| 操作 | activeProjectId | activeView | sidebarOpen | sidebarTab | dirtyFiles |
|---|---|---|---|---|---|
| 打开项目 | ← set | translate | true | null | [] |
| 关闭项目 | null | home | false | null | [] |
| 在校对审核中修改文件 | 不变 | 不变 | 不变 | 不变 | ← 追加路径 |
| ctrl+s 保存 | 不变 | 不变 | 不变 | 不变 | ← 移除路径 |

### 5.4 dirty 管理

```ts
// 标记文件为有未保存修改
function markDirty(filePath: string) {
  setAppState('dirtyFiles', files => [...new Set([...files, filePath])]);
}

// 标记文件为已保存
function markClean(filePath: string) {
  setAppState('dirtyFiles', files => files.filter(f => f !== filePath));
}
```

Tab 标题或文件树条目上以 `●`（红点）或 `M`（修改标记）显示 dirty 状态。

### 7.1 工具链

| 层 | 工具 | 说明 |
|---|---|---|
| 运行器 | **Vitest** | 与 Vite 同一配置，零额外开销 |

| 覆盖率 | **vitest --coverage**（@vitest/coverage-v8） | — |

### 7.2 测试范围

| 优先级 | 测试什么 | 怎么测 | 备注 |
|---|---|---|---|
| P0 | **appStore** 状态模型 | 纯函数测试 action（`openProject` / `closeProject` / `navigateTo`） | 无 DOM，vitest 直接跑 |
| P0 | **lib/api** 接口层 | 单测每个函数（原 React 版已有，**原样保留**） | 已验证 0 处 React 引用，直接搬 |
| P1 | **ToastHost / ConfirmHost** | 渲染测试 + 交互测试（点击 × 关闭、Enter/Escape） | @solidjs/testing-library |
| P1 | **ConnectionManager** | 模拟 `ensureDesktopBackendReady` 成功/失败，断言状态转换 | vi.fn() mock |
| P2 | 布局组件（ActivityBar / Sidebar / TitleBar） | 点击测试（tab 切换后 sidebarTab/activeView 是否正确） | — |
| P2 | 各页面组件快照 | 不追求全覆盖，关键路径测即可 | — |

### 7.3 vitest 配置要点

原 `vite.config.ts` 的 `test` 字段改：

```ts
/// <reference types="vitest" />
import { defineConfig } from 'vite';
import solidPlugin from 'vite-plugin-solid';

export default defineConfig({
  plugins: [solidPlugin()],
  test: {
    environment: 'jsdom',
    globals: true,
    setupFiles: ['./src/test-setup.ts'],   // 原文件保留，微调即可
```
