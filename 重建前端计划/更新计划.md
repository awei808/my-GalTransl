# 完全重建前端，重建前端审美、设计、使用体验
- 舍弃react，采用Tauri + Rust + SolidJS
- 可保留文件：接口层文件，纯逻辑文件，svg图标绘制文件
- 状态管理使用createStore
- 统一右下角弹窗负责处理信息反馈
- 操作体验应该要类似vscode，但用不着vscode的太多功能
- 基本页面：
    1. 顶部留一排按钮（文件、帮助、重做、撤回等），点击出现下拉菜单，项目级功能接口放在这里
    2. 页面左侧留一列按钮（翻译控制台、校对审核、查找替换、问题检测、设置等），是处理翻译项目的常用界面切换按钮；点击可激活侧边栏或切换主界面
    3. 剩余部分为主界面，依托状态转换来切换页面
    4. 整体风格：蓝色（浅色 #0066CC；深色 #0099FF）为主色调，浅色填充色 #F0F5FF，深色填充色 #12161C；禁用emoji，采用svg绘制简洁清爽的图标，可参考vscode的图标；采用圆角矩形，圆角半径使用参考8px网格系统
    5. 主界面类似vscode的代码编写界面，可自由调整长宽，侧边栏也可自由调整宽度
    6. 主界面主要承接翻译接口界面，也可切换为设置界面，校对审核界面；默认为首页（只显示logo、制作人、项目地址、精简帮助信息）
    7. 翻译控制台界面只有项目被打开时，才会出现；具体界面：顶部一个长宽比4:1的圆角矩形（矩形长度略小于主页面长度），矩形内左2/3显示当前项目的总行数、翻译进度、流程完成情况等基本数据，右1/3显示一个启动流程按钮（有下拉菜单选择翻译后端）；剩下界面分为两栏，可自由调整两栏的宽度：左栏为当前提示词显示，右栏为译文拼接显示。
    8. 设置界面通过点击页面左侧设置按钮的方式，切换主界面实现，包括系统设置和翻译项目设置，简单可视化；读取后端系统和翻译项目配置文件，显示参数名称，将参数对应的注释作为简要解释（可选），层级按照正常方法处理
    9. 校对审核界面只有打开项目文件时，才会出现；通过点击页面左侧审核校对按钮，再打开侧边栏对应文件的方式，切换主界面实现；校对审核页面主要实现一个文本编辑器功能，外加只显示json对象的特定属性功能、跳转至特定编号json的功能、增删修改当前json对象特定属性的快捷按钮功能
- 键盘快捷操作：至少保留ctrl+z/x/c/v/f/h/a/s，与vscode的对应功能一致；其中ctrl+f/h打开的是右上角的文件内查找替换功能，而不是左侧按钮的文件夹内全文件的查找替换功能
- 要有一个确认弹窗对话类，用于在缺少非关键数据时向用户确认是否执行操作，弹窗内部可嵌入html
