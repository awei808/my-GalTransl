完整翻译流程
入口层

__main__.py: main()
  └─ worker(project_dir, config.yaml, translator)
       └─ Service.py: run_job(JobSpec)
            └─ run_job_async()
                 ├─ 校验 JobSpec（project_dir / config_file_name / translator 非空）
                 ├─ CProjectConfig(project_dir, config.yaml)    ← 加载配置
                 ├─ 可选: 注入 backend_profile / prompt_template_overrides
                 └─ Runner.py: await run_galtransl(cfg, translator, stop_event)
Runner.py — 运行环境搭建

run_galtransl(cfg, translator, stop_event)
  │
  ├─ 1. 日志隔离（_JobThreadFilter + StreamHandler，防止并发 job 日志重复）
  ├─ 2. 可选: 定时启动（等 start_time 到达）
  ├─ 3. 创建目录: gt_input / gt_output / transl_cache
  ├─ 4. 加载插件: GTextPlugin（文本处理） + GFilePlugin（文件 I/O）
  ├─ 5. 初始化代理池 CProxyPool → 逐代理测活
  ├─ 6. 初始化 Token 池 COpenAITokenPool → 逐 token 测活
  ├─ 7. 初始化分割器: DictionaryCountSplitter / EqualPartsSplitter
  │
  └─ Frontend/LLMTranslate.py: await doLLMTranslate(cfg)
LLMTranslate.py — 主编排器

doLLMTranslate(cfg)
  │
  ├─ 读取配置: workersPerProject / sortBy / language / 字典路径 / splitFile
  ├─ get_file_list(input_dir) → 自然排序 → file_list
  │
  ├─ 线程池并发加载文件（最多 8 线程）
  │   └─ fplugins_load_file(file_path, fPlugins)
  │       └─ 轮流尝试每个 GFilePlugin.load_file()
  │       └─ 返回 (json_list, save_func)
  │
  ├─ 每个文件: input_splitter.split(json_list, file_path)
  │   └─ 产出一组 SplitChunkMetadata（含 trans_list、runtime_index）
  │   └─ total_chunks 累积
  │
  ├─ 短路判断（按 translator 类型分支）:
  │
  │   ┌─ "ForGal-full-pipeline" ──────────────────────────────────┐
  │   │   await _run_full_pipeline(cfg, file_json_lists, file_list) │
  │   │   return True                                               │
  │   └────────────────────────────────────────────────────────────┘
  │
  │   ┌─ "dump-name" → dump_name_table_from_chunks() → return
  │   ├─ "GenDic" → ensure_model_available() → gptapi.batch_translate(all_jsons) → return
  │   ├─ "ForFileMetaData" → gptapi.batch_translate(jsons, filename) 逐文件 → return
  │   ├─ "ForBatchMetaData" → gptapi.batch_translate(jsons, filename) 逐文件 → return
  │   │
  │   └─ "ForGal-json-multi-chat"（默认）→ 继续下面的翻译流程
  │
  ├─ chunk 排序（sortBy: name / size）
  ├─ name 替换表自动生成（首次运行） + 加载
  ├─ 字典加载: CNormalDic(pre) / CNormalDic(post) / CGptDict(gpt)
  ├─ init_gptapi(cfg) → 实例化对应后端类
  │
  └─ 翻译阶段:
      ├─ terminal_progress 进度条
      ├─ asyncio.Queue + worker 协程池（workersPerProject 个）
      ├─ 自适应并发调节（auto_tune_workers 后台任务，基于 429 比例和延迟）
      └─ 每个 worker: doLLMTranslSingleChunk()
单 Chunk 翻译

doLLMTranslSingleChunk(semaphore, split_chunk, cfg, gptapi)
  │
  ├─ async with semaphore（并发控制）
  │
  ├─ preprocess_trans_list()
  │   └─ 每个 CSentense:
  │       ├─ plugin.before_src_processed(tran)    ← 文本插件钩子
  │       ├─ tran.analyse_dialogue()              ← 检测「」括号、标记对话/独白
  │       ├─ pre_dic.do_replace(tran.post_src)    ← 译前字典替换
  │       └─ plugin.after_src_processed(tran)     ← 文本插件钩子
  │
  ├─ get_transCache_from_json(trans_list, cache_path)
  │   └─ 返回 (translist_hit, translist_unhit)
  │       缓存键 = 前句+当前句+后句 的上下文三元组
  │
  ├─ if translist_unhit:
  │   ├─ ensure_model_available_if_needed()
  │   ├─ 注入 file_metadata（如果后端支持 set_file_metadata）
  │   └─ gptapi.batch_translate(filename, cache_path, trans_list, num_per_request, ...)
  │       └─ 内部循环: 按 num_per_request 分批 → gptapi.translate() → 写缓存
  │
  ├─ postprocess_trans_list()
  │   └─ 每个 CSentense:
  │       ├─ plugin.before_dst_processed(tran)
  │       ├─ tran.recover_dialogue_symbol()       ← 恢复「」括号
  │       ├─ post_dic.do_replace(tran.post_dst)   ← 译后字典替换
  │       └─ plugin.after_dst_processed(tran)
  │
  └─ 标记 chunk 完成 → 如果该文件所有 chunk 完成:
      └─ postprocess_results()
          ├─ find_problems(trans_list, cfg, gpt_dict)  ← 12 种问题检测
          ├─ save_transCache_to_json(post_save=True)    ← 合并 .append.jsonl → .json
          └─ DictionaryCombiner.combine()               ← 合并 chunk → 套 name_replaceDict
              └─ save_func(output_path, result)         ← GFilePlugin 写输出文件
ForGal-full-pipeline 完整流水线

_run_full_pipeline(cfg, file_json_lists, file_list)
  │
  ├─ 阶段 0: DataValidator.validate_input_json()        ← 逐文件校验
  │
  ├─ 阶段 1: TextCompressor.compress()                   ← JSON 结构剥离 + 重复折叠
  │   └─ verify_compression()                           ← 校验完整性
  │
  ├─ 阶段 2: ForGlobalPrompt.batch_translate()
  │   ├─ _build_input_text_from_compressed()             ← 合并各文件压缩文本
  │   ├─ _build_glossary_text()                          ← 加载 GPT 字典
  │   ├─ _build_prompt_request()                         ← 填入 FORGLOBAL_PROMPT 模板
  │   ├─ ask_chatbot()                                   ← 调 LLM（仅 user 消息，无 system）
  │   ├─ _parse_global_prompt()                          ← 从返回值提取 JSON
  │   ├─ _normalize_global_prompt()                      ← 规整字段类型
  │   ├─ validate_global_prompt()                        ← DataValidator 校验
  │   └─ _save_global_prompt()                           → pass0_cache/GlobalPrompt.json
  │
  ├─ 阶段 3: GenDic.batch_translate(all_jsons)           ← 术语表（可跳过）
  │
  ├─ 阶段 4: ForFileMetaData.batch_translate()           ← 逐文件
  │   └─ 注入 GlobalPrompt 上下文（通过 [global_prompt] 占位符）
  │   → pass1_cache/FileMetaData.json
  │
  ├─ 阶段 5: ForBatchMetaData.batch_translate()          ← 逐文件
  │   └─ 注入 GlobalPrompt + FileMetaData 上下文
  │   → pass2_cache/BatchMetadata.json
  │
  └─ 阶段 6: _run_translation_phase()                    ← 与 doLLMTranslate 翻译阶段一致
      └─ 加载文件 → 切块 → worker 池 → ForGalJsonMulitChat.translate()
          └─ 首轮注入: GlobalPrompt + FileMetaData + BatchMetadata
          └─ 后续轮: 仅 jsonline + 术语表，多轮上下文自动携带
ForGalJsonMulitChat.translate() — 翻译核心

translate(trans_list, gptdict, proofread, filename)
  │
  ├─ _build_input_jsonlines()
  │   └─ 每句: sig|{"id": N, "name": "说话人", "src": "原文"}
  │       换行符 → <br>, tab → [t], 整批统一 n_symbol
  │
  ├─ _resolve_batch_metadata() + _trans_global_range()
  │   └─ 查当前批句子的全局行号区间对应的批次元数据
  │
  ├─ 重试循环（最多 3 次）:
  │   └─ _build_round_user_content()
  │       ├─ 首轮: 完整提示词 + GlobalPrompt + FileMetaData + BatchMetadata + 原文
  │       └─ 续轮: 仅 jsonline + 术语表
  │   └─ ask_chatbot(messages=完整历史, stream=True)
  │       └─ 流式回调: _parse_stream_lines() 边收边解析
  │   └─ _handle_parse_result()
  │       ├─ 切 </think> + 代码块提取 + sig 定位 + 引号修复
  │       ├─ 逐行: _parse_jsonline_result_line()
  │       │   ├─ 校验 sig 匹配（防串行）
  │       │   ├─ 校验 id 对应 trans_list[i].index
  │       │   ├─ 校验 dst 非空、无乱码
  │       │   └─ _normalize_parsed_translation_text()  ← OpenCC + 符号还原
  │       └─ 成功 → 追加 assistant 回复到 conversations[filename]
  │       └─ 失败 → _append_parse_failure_fallback_results() ← 标 (Failed)
BaseTranslate.ask_chatbot() — API 调用层

ask_chatbot(messages, stream, temperature, ...)
  │
  ├─ token 选择: random（随机轮询）/ fallback（主备切换）
  ├─ _wait_for_global_rpm_slot()                  ← 全局 RPM 限速
  ├─ client.chat.completions.create()             ← API 调用
  │   └─ 0.5s 间隔轮询 stop_event（支持取消）
  │
  ├─ 流式模式:
  │   └─ async for chunk in response:
  │       ├─ 检查 stop_event
  │       ├─ 收集 reasoning_content + content
  │       └─ stream_line_callback(lines, is_final)  ← 增量解析
  │
  ├─ 非流式模式:
  │   └─ response.choices[0].message.content
  │
  └─ 异常处理:
      ├─ RateLimitError → 标记限速 + 指数退避重试（2^try_count 秒 + jitter）
      ├─ 其他错误 → 记录 API Error + 退避重试
      └─ max_retry_count 耗尽 → RuntimeError
数据流总览

gt_input/*.json
  │
  ├─ GFilePlugin.load_file() → [{name, message}, ...]
  │
  ├─ Loader.load_transList() → CTransList（CSentense 双向链表）
  │
  ├─ CSplitter.split() → [SplitChunkMetadata, ...]
  │
  ├─ 缓存命中检查（三元组上下文键）→ hit / unhit
  │
  ├─ unhit → gptapi.translate() → LLM API
  │   └─ 解析 → 校验 → pre_dst 填入 CSentense
  │
  ├─ postprocess → post_dst
  │
  ├─ find_problems() → CSentense.problem
  │
  ├─ save_transCache_to_json() → transl_cache/pass3_cache/*.json
  │
  └─ DictionaryCombiner + save_func → gt_output/*.json
